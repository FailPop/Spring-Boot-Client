Информация об анализе

Здесь вы можете узнать о различных методах анализа кода, которые мы используем, и другой полезной информации.

Мы используем следующие правила анализа кода:

  • Проверка наличия табуляций в каждой строке
  • Проверка длины метода (не более 60 строк)
  • Проверка наличия пробелов после запятой, точки с запятой, приведения типов и условного оператора if
  • Проверка наличия пустых блоков кода
  • Проверка использования недопустимых типов
  • Проверка именования типов
  • Проверка наличия тега @since
  • Проверка наличия пробелов вокруг операторов присваивания
  • Проверка количества возвращаемых значений (не более 2 для не void методов и не более 1 для void методов)
  • Проверка порядка модификаторов
  • Проверка наличия недопустимых исключений в блоке catch
  • Проверка наличия пустых блоков catch

Мы используем Spring-Boot версии 3.1.4 и Checkstyle версии 8.42 для анализа кода на Java. Анализ кода на C++, C# и Python будет доступен позже.
